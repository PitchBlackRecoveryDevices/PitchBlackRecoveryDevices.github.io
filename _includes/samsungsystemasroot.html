This device uses system-as-root in an A-only partition configuration.

If you intend to root your ({{ page.codename }}) device, be advised that Magisk will be installed to the recovery partition, not the usual boot partition. Both PBPR and Magisk-rooted Android boot using the recovery kernel, with the device's hardware keys used to choose between PBPR and Android at boot time.

You are therefore strongly advised to first root your device with Magisk and then pre-root a PBPR image before flashing it to recovery. The usual path of installing PBPR prior to rooting is problematic on this device, because the Magisk installer does not support installation to recovery from PBPR. Furthermore, prior rooting of the device with Magisk carries the significant advantage of disabling various device protection mechanisms that you will otherwise need to take care of manually.

If you wish to install PBPR without rooting, there are numerous pitfalls to consider. For example, a stock device is protected by Android Verified Boot. This is implemented as a vbmeta partition that contains cryptographic digests for verifying the integrity of boot.img, recovery.img and other partitions/images. Flashing PBPR without first flashing a vbmeta image with verity disabled will render your device unable to boot.

Please bear all of the above in mind when considering the installation methods listed below, and refer to the device's PBPR support thread on XDA for detailed information about installation and the associated hazards. Details of the firmware version(s) with which the current PBPR image's kernel is compatible can be found in the PBPR changelog for this device.
